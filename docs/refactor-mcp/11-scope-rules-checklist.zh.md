## 1. 「口径」是什么？你要确认什么？

**口径 = 回答「功能/服务清单」时，到底把什么算进去、什么不算、怎么分组、怎么对外说。**

你不需要：
- 背每个服务干什么  
- 审每一条抽取结果是否「业务正确」  

你需要：
- 对下面 **选择题式规则** 选 A/B/C（或改一句话）  
- 这些规则会写进配置文件（如 `config/catalog-rules.yaml`），抽取器按规则跑  

**确认方式：** 直接回复本清单各题的选项字母即可（例如：`M1=A, M2=A, W1=A, …`）。  
有拿不准的标 `?`，我们先用推荐默认值并在答案里写明口径。

---

## 2. 规则文件长什么样（给你看形态，不是让你手写）

确认后会落成类似：

```yaml
# config/catalog-rules.yaml（示意）
middleware:
  microservice_list:
    source: edition-manifest.services   # 不用目录瞎扫当权威
    editions: merge_basic_and_advance   # 或分套回答
    exclude_kinds: [infrastructure]     # postgres 等不进微服务
    # exclude_names: [doc-toolkit]      # 可选黑名单

chatkit-web:
  app_list:
    source: package.json.workspaces
    libraries_in_app_list: false        # libs 单独「共享库」清单
  admin_feature_list:
    source: [routes, i18n_zh]
    exclude_paths: [/login, /]

finclaw:
  module_list:
    source: cargo_crates
    exclude: [vendor/*]
  cli_feature_list:
    source: clap_subcommands
```

你确认的是这些 **选项**；实现时写成配置，不是每次口头约定。

## 3. middleware 口径题（请选）

| 选项 | 含义 |
|------|------|
| **A（推荐）** | 只认 `edition-manifest.yaml` 里的 `services` |
| B | manifest ∪ 目录 `services/**` 扫描（目录多出来的标低置信） |
| C | 只认 docker-compose 里的业务服务名 |

你的选择：`M1 = ?`（默认按 A）

### M2. basic / advance 两套 edition

| 选项 | 含义 |
|------|------|
| **A（推荐）** | 默认答 **合并去重** 全量服务；若用户问「基础版/进阶版」再按 edition 过滤 |
| B | 默认只答 basic；advance 要用户显式问 |
| C | 永远分两段输出（Basic 列表 + Advance 增量） |

你的选择：`M2 = ?`

### M3. postgres / redis / nats 等 infrastructure

| 选项 | 含义 |
|------|------|
| **A（推荐）** | **不计入**「微服务清单」；若问「基础设施」另给清单 |
| B | 计入微服务清单但标注「基础设施」 |
| C | 一律不出现在任何清单 |

你的选择：`M3 = ?`

### M4. connector（飞书/钉钉/企微等）

| 选项 | 含义 |
|------|------|
| **A（推荐）** | 算进微服务清单（manifest 里有就保留） |
| B | 单独「通道/连接器」清单，主微服务清单里排除 |
| C | 对外 public 隐藏，仅 debug 可见 |

你的选择：`M4 = ?`

### M5. 问「功能清单」时 middleware 默认给什么

| 选项 | 含义 |
|------|------|
| **A（推荐）** | 默认 = 服务清单（M1 结果）；并说明「按部署服务枚举」 |
| B | 默认 = 服务 + flows（入站/出站）两节 |
| C | 默认 = 服务 + flows + OpenAPI tags（可能较长） |

你的选择：`M5 = ?`
